## вңҰ What Is This?

**Unbiased AI** is a full-stack fairness monitoring platform that plugs into your ML models and answers one critical question:

> *"Is my AI treating everyone equally?"*

It combines a **FastAPI backend** with trained scikit-learn models, a **React + TypeScript frontend**, and a suite of bias detection and mitigation algorithms вҖ” all wrapped in a clean, dark-mode-ready dashboard.

## вңҰ Quick Start

### Prerequisites

| Tool | Version |
|------|---------|
| Python | вүҘ 3.11 |
| Node.js | вүҘ 20 |
| npm | вүҘ 10 |

### 1 В· Clone & Bootstrap

```bash
git clone https://github.com/your-username/unbiased-ai.git
cd unbiased-ai

# Generate placeholder ML models (required before first run)
python create_dummy_models.py
```

### 2 В· Backend

```bash
cd backend

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp ../.env.example .env
# Edit .env вҖ” Firebase or MONGO_URL is optional; leave blank for JSON fallback

# Start the API server
uvicorn main:app --reload --port 8000
```

> API docs available at **http://localhost:8000/docs**

### 3 В· Frontend

```bash
cd frontend

# Install dependencies
npm install

# Configure API endpoint
cp .env.example .env.local
# VITE_API_BASE_URL=http://localhost:8000

# Start dev server
npm run dev
```

> App available at **http://localhost:5173**

## вңҰ Fairness Metrics Explained

| Metric | Formula | Threshold | Meaning |
|--------|---------|-----------|---------|
| **Demographic Parity Difference** | P(Е¶=1\|A=a) вҲ’ P(Е¶=1\|A=b) | < 0.10 ideal | Equal positive rates across groups |
| **Equal Opportunity Difference** | TPR_a вҲ’ TPR_b | < 0.10 ideal | Equal true-positive rates for qualified candidates |
| **Disparate Impact Ratio** | P(Е¶=1\|minority) / P(Е¶=1\|majority) | вүҘ 0.80 (80% rule) | Legal compliance threshold |
| **Bias Risk Score** | Composite 0вҖ“100 | < 40 = low risk | Internal risk band for flagging |

## вңҰ Deploy in 5 Minutes

### Backend вҶ’ Render

1. Push repo to GitHub
2. [New Web Service](https://render.com) вҶ’ connect repo
3. Render auto-detects `render.yaml` вҖ” no config needed
4. Add Firebase env vars (`FIREBASE_PROJECT_ID` + `FIREBASE_SERVICE_ACCOUNT_JSON`) or `MONGO_URL`
5. Copy the deployed URL

### Frontend вҶ’ Netlify

1. [New Site](https://netlify.com) вҶ’ Import from Git
2. Netlify auto-detects `netlify.toml`
3. Add env var: `VITE_API_BASE_URL=https://your-render-url.onrender.com`
4. Deploy рҹҡҖ
